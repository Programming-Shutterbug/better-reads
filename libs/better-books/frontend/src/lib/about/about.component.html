Projectcasus Betterreads

Deze webapplicatie is gemaakt in het kader van het vak Client-side web programming individual

Voor deze casus heb ik gekozen voor 'Betterreads'. In deze applicatie kan de gebruiker bijhouden welke boeken ze gelezen hebben, welke ze nog willen lezen etc. De gebruikers kunnen nieuwe boeken ook zelf toevoegen, de boeken die ze gelezen hebben kunnen ze beoordelen. Daarnaast zijn er ook nog schrijvers, deze zijn te volgen. Gebaseerd op andere gebruikers die deze schrijver volgen komt er een aanbeveling uit voor een nieuwe schrijfer.

Functional requirements

De applicatie biedt de onderstaande functionaliteiten:

ID Requirements Toelichting Moscow Prioriteit
1 Inloggen De gebruiker moet inloggen/registeren om de inhoud van de website te bekijken. Must
2 Registreren De gebruiker moet kunnen registreren om een account aan te maken zodat de inhoud van de website te zien is. Must
3 Lijst van eigen boeken weergeven De gebruiker begint met een lijst van zijn eigen gelezen, nog te lezen, DNF boeken. Must
4 Lijst van boeken weergeven De gebruiker kan naar 'Boeken' navigeren waar alle boeken staan, waar de gebruiker in kan zoeken en filteren. Must
5 Nieuw boek toevoegen De gebruiker kan bij 'Boeken' een nieuw boek toevoegen. Must
6 Detailpagina boek inzien Via de lijst van boeken kan de gebruiker op een boek klikken en naar de detailpagina van een boek gaan voor meer informatie. Must
7 Tags geven De gebruiker kan via de detailpagina van een boek een tag geven: gelezen, nog te lezen, DNF. Must
8 Beoordelen Als de gebruiker de tag 'gelezen' heeft gebruikt, krijgt de gebruiker een pagina/pop-up om het boek te beoordelen. Must
9 Beoordelen lijst De gebruiker kan naar 'Beoordelingen' navigeren waar alle beoordelingen staan, waar de gebruiker in kan zoeken en filteren. Could
10 Lijst van schrijver Er is een lijst met 'Schrijvers' waar alle schrijvers in staan, waar de gebruiker in kan zoeken en filteren. Must
11 Detailpagina schrijver Detailpagina opent als er op een schrijver in de lijst wordt geklikt door de gebruiker. Should
12 Schrijver volgen De schrijver kan gevolgd worden door de gebruiker. Must
13 Aanbeveling nieuwe schrijver Als er een schrijver gevolgd wordt, komt er een aanbeveling gebaseerd op de andere gebruikers die de schrijver volgen. Must
14 Gebruiker wijzigen gegevens Gebruiker kan in hun account hun eigen gegevens wijzigen. Should

Non-fuction requirements

De applicatie biedt de onderstaande non-function requirements:

ID Requirements Toelichting
1 CI/CD pipeline Uitgebreid versiebeheer met feature branching toegepast.
2 Security en authenticatie Afhankelijk van de rol heeft een user toegang tot functionaliteit.
3 Useability Opvallend gebruiksvriendelijk systeem en eigen vormgeving passend bij casus.
4 Tests Naast unittesten ook end-to-end tests voor 2 scenario's (anders dan authenticatie)
5 Documentatie Requirements en/of userstories zijn helder en volledig gedocumenteerd.
6 Data integrity (validatie van data in UI) Uitgebreide validatie van alle inputvelden via Reactive Form Validation.
